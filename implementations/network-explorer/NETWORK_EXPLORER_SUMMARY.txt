PRISM NETWORK EXPLORER — ONE-PAGE SUMMARY
================================================================================

WHAT IT SHOWS:
  A real-time graph visualization of criminal networks extracted from FIR records.
  Every accused person is a node. Every co-appearance in the same FIR creates an edge.

WHY IT MATTERS:
  Excel sheets show crime statistics. This page shows STRUCTURE.
  It reveals which offenders work together, who the kingpins are, and how 
  organized each criminal group is.

THE FOUR MAIN PANELS:
  
  ┌─────────────────────────────────────────────────────────────────┐
  │ FILTERS (left)              GRAPH (center)          DETAILS (right)
  │                                                                   │
  │ • Search by name            Force-directed graph    • Name, Age   │
  │ • Crime type                showing accused nodes   • Risk Score  │
  │ • District                  connected by shared     • FIR count   │
  │ • Date range                FIRs                    • Associates  │
  │                             (edge thickness =       • MO          │
  │ LEGEND:                     co-appearance count)    • Timeline    │

  │                             with gang labels]        when node    │

  └─────────────────────────────────────────────────────────────────┘

THE THREE VIEWS (tabs at top):

  1. ALL CONNECTIONS (default)
     Shows every node and edge. Highest density. Looks complex but real.
  
  2. GANG CLUSTERS
     Same graph, but nodes color-coded by Louvain community detection.
     Each suspected gang circled and labeled.
  
  3. REPEAT OFFENDERS
     Filters to only accused with 2+ FIRs. Edges show co-appearance counts.

WHAT HAPPENS WHEN YOU INTERACT:

  Click search box            → Autocomplete suggests accused names
  Type a name                 → Graph filters to that person's network
  Click crime type dropdown   → Graph shows only that crime type's network
  Select a date range         → Re-fetches graph for that time period
  Click a node                → Detail panel pops up on the right
  Hover over edge             → Tooltip: "Co-accused in 3 FIRs"
  Click "Gang Clusters" tab   → Nodes recolor, cluster circles appear

DATA SOURCES:

  Data comes from three tables:
  
  1. Accused (who)
     - Name, age, gender, FIR count
     - Is this person still absconding? 
  
  2. CaseMaster (what happened)
     - When did each FIR occur (date/time)
     - Which accused were in it (co-accused relationship)
  
  3. ArrestSurrender + ChargesheetDetails (outcome)
     - Was accused arrested?
     - Was case chargesheeted?

THE INTELLIGENCE INSIGHT:

  "One FIR has 2 accused. Another FIR from 3 months later has the same 2 people.
  A third FIR has one of them with a new partner. That new partner worked with 
  someone else in another FIR. The algorithm asks: Are these 6 people organized?
  
  Louvain community detection says: Yes, these 5 are tightly linked (same gang).
  One outlier only co-appeared once with the group.
  
  Centrality analysis says: Person #2 is the kingpin (highest betweenness).
  All information flows through them."

BACKEND HEAVY LIFTING:

  1. Query co-accused relationships from 1000+ FIRs
  2. Fuzzy-match accused names (resolve "Abdul Rauf" = "A. Rauf Khan" = same person)
  3. Build NetworkX graph object
  4. Run Louvain algorithm (identifies clusters/gangs)
  5. Compute centrality metrics (who's important?)
  6. Serialize to JSON → send to frontend

FRONTEND RENDERING:

  1. D3.js force-directed simulation
     - Nodes repel each other
     - Edges pull nodes together
     - Result: natural circular clusters of related people
  
  2. Interactive picking
     - Hover over node → highlight connected nodes
     - Click node → fetch full profile, display in right panel
  
  3. Filtering
     - Crime type dropdown → re-render subgraph
     - Date range → re-fetch different time period

FILES TO BUILD (backend):

  backend/routers/network.py
    ├── GET /api/network/graph        — returns full graph JSON
    ├── GET /api/network/profile/{id} — returns accused profile
    └── GET /api/network/search       — fuzzy search accused

  backend/agents/network_agent/
    ├── graph_builder.py              — queries DB, builds graph
    ├── entity_resolver.py            — fuzzy matching for repeat offenders
    ├── community_detection.py        — runs Louvain
    └── centrality.py                 — computes betweenness/degree

FILES TO BUILD (frontend):

  frontend/src/components/network/FilterPanel.tsx  — left sidebar
  frontend/src/components/network/NodeDetailPanel.tsx — right sidebar
  frontend/src/hooks/useNetworkGraph.ts            — data fetching
  frontend/src/services/network.service.ts         — API calls

TECHNICAL CHALLENGES:

  1. Performance: 500+ nodes = slow D3. Solution: pre-compute 300 iterations.
  2. Name matching: 3 rows same person. Solution: fuzzy matching + age/gender.
  3. Clustering: Louvain is slow. Solution: cache in Redis, run on Cron schedule.
  4. React + D3: Both mutate DOM. Solution: D3 touches SVG, React manages state.
