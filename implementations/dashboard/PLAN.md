The dashboard (`CommandDashboardScreen.tsx`) already has a **complete, high-quality UI skeleton** that matches the spec. However, it is entirely driven by mock/hardcoded data:

| Section | Current State | Target |

| **KPI Strip (4 cards)** | Returns from a static `getKpis()` switch with 3 hardcoded districts | Real counts from `GET /api/dashboard/stats` |

**Backend state:** `backend/main.py` currently only registers 3 test/seed routers. There are **no production routers, schemas, or analytics modules** yet — everything documented in `DASHBOARD_SPEC.md` is greenfield.

## 1. Prerequisites & Missing Pieces

These must exist or be created **before or during** implementation. They are blockers.

### 1.1 Backend Infrastructure (Critical Blockers)

#### `backend/core/` — Entirely Missing
The entire `core/` layer described in `project-structure.md` does not exist yet:
- **`core/database.py`** — DB session factory. The project uses **Zoho Catalyst DataStore** (not PostgreSQL/SQLAlchemy). The actual DB access pattern uses `zcatalyst_sdk` with a `datastore()` object, as seen in `tests/db/data_insertion.py`. A `get_datastore(request)` dependency function needs to be extracted here.
- **`core/security.py`** — `require_role()` FastAPI dependency that validates the Catalyst session and extracts the user's role. Without this, all `Depends(require_role(...))` calls in the spec fail.

> **Critical Note on DB:** The spec references raw SQL syntax, but the actual data layer uses
> the **Zoho Catalyst DataStore SDK** (a hosted NoSQL/ZCQL datastore). Queries must be adapted
> from raw SQL to Catalyst's own query API (ZCQL or column search), with grouping and aggregation
> done in Python when the DB doesn't support it natively.

#### `backend/routers/` — Entirely Missing
No production routers exist. `main.py` only includes 3 test routers.

#### `backend/schemas/` — Entirely Missing
No Pydantic response models exist yet.

#### `backend/analytics/` — Entirely Missing
No analytics logic modules exist.

#### Derived Tables in Catalyst DataStore — May Not Exist
The spec requires these tables:
- **`dashboard_stats`** — precomputed KPI row written nightly. Must be created in Catalyst console.
- **`crime_alerts`** — stores generated spike alerts. Must be created.
- **`risk_scores`** — per-accused risk score rows. Must be created.

If these tables don't exist, the real-DB endpoints will fail. They need to be created and seeded with at least one stub row for the initial phase.

### 1.2 Frontend Infrastructure (Partially Missing)

#### `frontend/src/services/` — Does Not Exist
The `services/` directory is in the spec but has not been created. No `dashboard.service.ts` exists.

#### `frontend/src/hooks/useDashboardStats.ts` — Does Not Exist
No custom hooks directory or file exists yet.

#### `frontend/src/lib/api-client.ts` — Does Not Exist
The axios instance with interceptors does not exist. The frontend currently makes no real API calls.

#### `axios` — Not Installed
`package.json` does not include `axios`. It must be installed before `api-client.ts` can work.

#### `frontend/src/types.ts` — Needs Extension
The existing `Alert` type uses `level: 'CRITICAL' | 'ELEVATED' | 'INFO'`. The spec's `AlertItem` uses `level: 'CRITICAL' | 'WARNING'`. The type needs to be updated or a new dashboard-specific type added.

### 1.3 Data Availability

#### Seed Data May Be Insufficient for Dashboard
The `data_insertion.py` script inserts 16 cases (4x scale). This may produce zero counts for:
- `dashboard_stats.active_alert_count` (if no alerts were generated by the alert engine)
- `risk_scores` (if the scoring job has never run)
- Trend sparklines (need 7 weeks of spread `IncidentFromDate` values)

A **stub/hardcoded response phase** (Step 3 in the order below) is mandatory to unblock frontend work while real data is verified.

### Step 7 — Create `useDashboardStats.ts`

#### CREATE `frontend/src/hooks/useDashboardStats.ts`

All 4 hooks in one file, per spec. Each follows the same `{ data, loading, error }` pattern.

```typescript
import { useState, useEffect } from 'react';
import {
  dashboardService,
  DashboardStats,
  DistrictCrimeData,
  AlertItem,
  TrendData,
} from '../services/dashboard.service';

// Hook 1: KPI stats — fetches once on mount
export function useDashboardStats() {
  const [stats, setStats] = useState<DashboardStats | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    dashboardService.getStats()
      .then(setStats)
      .catch(() => setError('Failed to load stats'))
      .finally(() => setLoading(false));
  }, []);

  return { stats, loading, error };
}

// Hook 2: District crimes — re-fetches whenever timeframe changes
export function useDistrictCrimes(timeframe: '24h' | '7d' | '30d') {
  const [data, setData] = useState<DistrictCrimeData[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setLoading(true);
    dashboardService.getDistrictCrimes(timeframe)
      .then(setData)
      .finally(() => setLoading(false));
  }, [timeframe]);

  return { data, loading };
}

// Hook 3: Active alerts — fetches on mount, then polls every 60 seconds
export function useAlerts() {
  const [alerts, setAlerts] = useState<AlertItem[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    dashboardService.getAlerts().then(setAlerts).finally(() => setLoading(false));

    const interval = setInterval(() => {
      dashboardService.getAlerts().then(setAlerts);
    }, 60_000);

    return () => clearInterval(interval);
  }, []);

  return { alerts, loading };
}

// Hook 4: Crime trend sparklines — fetches once on mount
export function useTrends() {
  const [trends, setTrends] = useState<TrendData[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    dashboardService.getTrends()
      .then(setTrends)
      .finally(() => setLoading(false));
  }, []);

  return { trends, loading };
}
```

#### 8.4 Crime Density Map — Dynamic Pins

Replace the 3 hardcoded `<button>` pins with a `.map()` over `districts`:

```tsx
// Karnataka bounding box for lat/lng → % position projection:
const KA_LAT_MIN = 11.6, KA_LAT_MAX = 18.5;
const KA_LNG_MIN = 74.0, KA_LNG_MAX = 78.6;
const project = (lat: number, lng: number) => ({
  left: ((lng - KA_LNG_MIN) / (KA_LNG_MAX - KA_LNG_MIN)) * 100,
  top:  ((KA_LAT_MAX - lat)  / (KA_LAT_MAX - KA_LAT_MIN)) * 100,
});

// Dot color based on relative crime intensity:
const maxFirs = Math.max(...districts.map(d => d.total_firs), 1);
const getIntensity = (count: number) => {
  const ratio = count / maxFirs;
  if (ratio > 0.7) return { dot: 'bg-red-500 border-red-500',     label: 'text-[#ff4d4d]'  };
  if (ratio > 0.4) return { dot: 'bg-orange-500 border-orange-500', label: 'text-orange-400' };
  return             { dot: 'bg-yellow-500 border-yellow-500',   label: 'text-yellow-400' };
};

// JSX:
{mapLoading ? null : districts.map((district) => {
  const pos = project(district.lat, district.lng);
  const intensity = getIntensity(district.total_firs);
  const isActive = selectedDistrict?.district_id === district.district_id;
  return (
    <button
      key={district.district_id}
      onClick={() => setSelectedDistrict(isActive ? null : district)}
      className="absolute group cursor-pointer"
      style={{ left: `${pos.left}%`, top: `${pos.top}%` }}
    >
      <span className={`absolute -translate-x-1/2 -translate-y-1/2 w-4 h-4 rounded-full border
        flex items-center justify-center ${intensity.dot}
        ${isActive ? 'scale-125 shadow-[0_0_12px_rgba(239,68,68,0.8)]' : 'hover:scale-110'}
        transition-transform`}>
        <span className="w-1.5 h-1.5 rounded-full bg-current animate-ping" />
      </span>
      <span className={`absolute left-3 -top-3 bg-black border border-white/10 text-[9px]
```

```python
def normalize(counts: list[int]) -> list[int]:
    """Normalize weekly counts to 0–100 scale for bar height rendering."""
    max_val = max(counts) if max(counts) > 0 else 1
    return [int((c / max_val) * 100) for c in counts]

def compute_trend(current: int, prior: int) -> tuple[float, str]:
    """Compute % change and trend direction string."""
    if prior == 0:
        return (0.0, "flat")
    change = ((current - prior) / prior) * 100
    trend = "up" if change > 2 else "down" if change < -2 else "flat"
    return (round(change, 1), trend)

def format_time_ago(created_at: datetime) -> str:
    """Convert a datetime to a human-readable 'Xh ago' string."""
    diff = datetime.utcnow() - created_at
    hours = int(diff.total_seconds() / 3600)
    if hours < 1:
        return f"{int(diff.total_seconds() / 60)}m ago"
    if hours < 24:
        return f"{hours}h ago"
    return f"{diff.days}d ago"
```

```python
# Compute cutoff date from timeframe param

since_iso = (datetime.utcnow() - timedelta(days=days)).isoformat()

# Fetch CaseMaster rows after cutoff
case_table = db.table("CaseMaster")
# Use ZCQL: SELECT * FROM CaseMaster WHERE IncidentFromDate >= :since
cases = case_table.search_by_column_name("IncidentFromDate", since_iso, ...)

# Fetch Unit and District lookup tables once (small tables, safe to load fully)
units = {row["ROWID"]: row for row in db.table("Unit").get_all_rows()}
districts = {row["ROWID"]: row for row in db.table("District").get_all_rows()}
crime_sub_heads = {row["ROWID"]: row for row in db.table("CrimeSubHead").get_all_rows()}

# Aggregate in Python: group cases by district, count totals
from collections import defaultdict, Counter
district_stats = defaultdict(lambda: {"total_firs": 0, "crime_types": []})
for case in cases:
    unit = units.get(case.get("PoliceStationID"))
    if not unit:
        continue
    dist = districts.get(unit.get("DistrictID"))
    if not dist:
        continue
    d_name = dist.get("DistrictName", "Unknown")
    district_stats[d_name]["total_firs"] += 1
    csh = crime_sub_heads.get(case.get("CrimeMinorHeadID"))
    if csh:
        district_stats[d_name]["crime_types"].append(csh.get("CrimeSubHeadName", ""))

# Build response with centroids
results = []
for d_name, d_data in district_stats.items():
    centroid = DISTRICT_CENTROIDS.get(d_name, {"lat": 15.0, "lng": 76.0})
    dominant = Counter(d_data["crime_types"]).most_common(1)
    results.append(DistrictCrimeResponse(
        district_id=hash(d_name) % 1000,
        district_name=d_name,
        total_firs=d_data["total_firs"],
        active_cases=0,  # TODO: join CaseStatusMaster
        high_risk_count=0,  # TODO: join risk_scores
        dominant_crime_type=dominant[0][0] if dominant else "Unknown",
        lat=centroid["lat"],
        lng=centroid["lng"],
    ))
return sorted(results, key=lambda r: r.total_firs, reverse=True)
```
